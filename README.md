# TeamUniverse Bot

Бот воронок TeamUniverse: магниты, тест «Подходишь ли ты», кастинг стримерш,
анкета франшизы, цепочки прогрева. Реализация по teamuniverse_bot_spec.md.

## Архитектура
GitHub (код) → Railway (запуск, редеплой по push) → Supabase (Postgres, постоянная база).
База ВНЕШНЯЯ и не зависит от деплоев: код можно передеплоивать сколько угодно, данные живут в Supabase.
Локально без DATABASE_URL бот автоматически работает на SQLite — для теста на своей машине.

## Файлы
bot.py — все сценарии · texts.py — все тексты (заглушки помечены [ЗАГЛУШКА]) ·
storage.py — хранилище (Postgres/SQLite автоматически) · config.py — настройки из переменных ·
Procfile — команда запуска для Railway · requirements.txt · .env.example — образец переменных.

## Запуск по шагам

### 1. Telegram (15 мин)
1. @BotFather → /newbot → сохранить токен. Там же /setcommands:
   menu - Главное меню
   casting - Кастинг стримерш
   materials - Мои материалы
   help - Связь с человеком
2. Добавить бота АДМИНОМ в канал (иначе не работает проверка подписки).
3. Создать рабочую группу заявок (вы + Денис), добавить бота.
4. Узнать id: свой и Дениса — через @userinfobot; id группы — переслать из неё
   сообщение @userinfobot (id группы отрицательный, вида -100...).

### 2. Локальный тест (30 мин, до деплоя!)
1. cp .env.example .env → заполнить BOT_TOKEN, CHANNEL_ID, id получателей.
   DATABASE_URL оставить ПУСТЫМ — локально работает SQLite.
2. Положить PDF в assets/ (antiscam_checklist.pdf, guide30.pdf).
3. pip install -r requirements.txt → python bot.py
4. Прогнать все двери: ?start=test (три вердикта), ?start=antiscam (без подписки и с ней),
   ?start=casting (возраст 17 → отказ; нормальный с кружком → карточка Денису),
   ?start=franchise (горячий → карточка), голый /start → меню и прослойки.

### 3. Supabase (10 мин)
1. supabase.com → New project (регион EU, пароль базы сохранить).
2. Settings → Database → Connection string → вкладка **Session pooler** → скопировать URI
   (host вида aws-0-...pooler.supabase.com:5432 — он работает по IPv4, прямое подключение
   на бесплатном тарифе только IPv6 и с Railway может не соединиться).
3. В URI подставить пароль. Это значение DATABASE_URL.
4. Таблицы создавать не нужно — бот создаёт их сам при первом старте (init_db).
5. Заявки смотреть: Supabase → Table Editor → casting / franchise. Статус заявки
   («у Дениса», «созвон», «отказ») менять прямо там в столбце status.

### 4. GitHub (10 мин)
1. Создать ПРИВАТНЫЙ репозиторий.
2. Залить все файлы проекта. Проверить, что .env НЕ попал (он в .gitignore).
   PDF магнитов тоже в .gitignore — их грузим не через git (см. ниже).

### 5. Railway (15 мин)
1. railway.app → вход через GitHub → New Project → Deploy from GitHub repo.
2. Variables → добавить: BOT_TOKEN, CHANNEL_ID, CHANNEL_URL, DENIS_CHAT_ID,
   ARTEM_CHAT_ID, WORK_CHAT_ID, ADMIN_IDS, DATABASE_URL (из шага 3).
3. PDF магнитов: простейший путь — убрать assets/*.pdf из .gitignore и закоммитить
   файлы в приватный репозиторий (допустимо: репозиторий закрытый). Альтернатива
   позже — раздача file_id через Telegram.
4. Deploy → смотреть логи: должно быть "TeamUniverse bot started".
5. Прогнать двери ещё раз на боевом боте. Проверить в Supabase Table Editor,
   что заявка появилась в таблице.

### 6. Рабочий цикл
Правка кода → commit → push в main → Railway передеплоит сам. База не затрагивается.

## Перед публичным запуском — чек-лист

- [ ] Вилки бюджета франшизы — bot.py, хендлер f_budget
- [ ] Мини-карточки типажей заменить на материалы Антона (сейчас v1)
- [ ] PDF магнитов — финальные, не болванки
- [ ] Экран согласия на перс. данные перед анкетами — после консультации юриста
- [ ] ?start=franchise НЕ публиковать до поста-моста (неделя 10)
